## What an "authoritative source" is here (read before fixing anything)
- **The court PDF/text content is reliable; the FILENAME and DB label may be wrong.**
  A systemic scraper file-naming scramble misfiled a set of opinions: a file named
  `YYYYNDn` can contain a *different* opinion. Always trust the cite printed *inside*
  the opinion, never the filename/DB label alone.
- **Canonical refs tree = `~/refs/nd/opin`** (per CLAUDE.md): `pdfs/<year>/<year>NDn.pdf`,
  `markdown/<year>/<year>NDn.md`, `archive/<year>/<docket>.htm`. (`~/cDocs/refs/opin/pdf`
  is a *separate* tree the user also uses — confirm before writing there.)
- **Westlaw `.doc`-sourced rows never restate the opinion's own neutral cite in the body**
  (it's only in metadata) → they false-positive in both the CITE and BODYDUP detector
  passes. Treat `.doc` source rows specially.

  render page 1 and read it visually (Read tool on a rendered PNG) to get the filing date.
